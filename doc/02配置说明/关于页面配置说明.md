# 👤 关于页面配置说明

本文档详细介绍 RouxZhee 中个人介绍页面（`/about`）的配置方式。关于页面采用**双层配置架构**：

- **默认配置**：`src/config/about.config.ts`
- **用户自定义配置**：`src/config/.config/about.config.ts`

用户配置会覆盖默认配置，建议通过用户配置文件修改内容，避免直接改动默认配置。

> 💡 **提示**：所有字段值都可以不使用引号包裹，除非包含特殊字符。

---

## 1️⃣ 配置架构

### 双层配置原理

```
src/config/about.config.ts        ← 默认配置（不建议直接修改）
src/config/.config/about.config.ts ← 用户配置（推荐在此修改）
```

构建时，系统会自动合并两个配置：

1. 优先读取 `src/config/.config/about.config.ts` 中的 `userAboutConfig`
2. 使用 `mergeConfig` 函数与默认配置合并
3. 导出最终的 `aboutConfig` 供页面组件使用

## 2️⃣ 首屏 · 身份锚定区（hero）

**说明**：设置页面顶部头像、名称、角色、价值主张和标签。

**配置项**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `avatar` | `string` | 博主头像图片地址 |
| `avatarAlt` | `string` | 头像 alt 文本 |
| `name` | `string` | 博主名称 |
| `handle` | `string` | 社交账号或昵称 |
| `role` | `string` | 身份角色描述 |
| `valueProposition` | `string` | 价值主张/一句话介绍 |
| `stats` | `array` | 核心数据展示 |
| `headerTags.left` | `array` | 头像左侧标签 |
| `headerTags.right` | `array` | 头像右侧标签 |

**示例**：

```typescript
hero: {
  avatar: 'https://example.com/avatar.avif',
  avatarAlt: '博主头像',
  name: '余温',
  handle: '@yu_wen',
  role: '资深云原生架构师 ｜ 前阿里 P7 ｜ 技术专栏作者',
  valueProposition: '用 8 年一线踩坑经验，帮你把复杂的系统架构，拆解成看得懂的实战笔记',
  stats: [
    { icon: '👨‍💻', value: '8年', label: '后端经验' },
    { icon: '📦', value: '千万级', label: '主导日活中台' },
    { icon: '📝', value: '156篇', label: '原创图解' },
  ],
  headerTags: {
    left: [
      '🤝 上知天文地理，下知鸡毛蒜皮',
      '🏃 拼尽全力终于没有力气',
    ],
    right: [
      '😏 说实话你挺好看的，但我没说实话',
      '🔍 Ta 只是喜欢你，又不是只喜欢你',
    ],
  },
}
```

## 3️⃣ 故事钩子 · 共鸣区（story）

**说明**：通过故事化叙述增强读者共鸣，`paragraphs` 数组可继续追加段落。

**配置项**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `image` | `string` | 故事配图路径 |
| `quote` | `string` | 金句/引语 |
| `paragraphs` | `array` | 故事段落数组，可追加 |
| `testimonial.author` | `string` | 读者反馈作者 |
| `testimonial.content` | `string` | 读者反馈内容 |

**示例**：

```typescript
story: {
  image: '/assets/shiroki.avif',
  quote: '我曾经也因为半夜线上告警，抱着电脑在出租屋里哭过',
  paragraphs: [
    '起点：我曾是个连 Git 都不会用的菜鸟',
    '转折：在无数次通宵加班和踩坑中，我总结出了一套自己的方法论',
    '现在：我把这些血泪教训写成文章，只为让后来的你，少熬一个夜',
    // 可继续追加更多故事段落
  ],
  testimonial: {
    author: '@白木',
    content: '看了你的 K8s 系列，我终于搞懂了 Pod 的生命周期',
  },
}
```

## 4️⃣ 硬核履历 · 时间轴（career）

**说明**：以时间轴形式展示职业经历，`items` 数组可继续追加经历。

**配置项**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `title` | `string` | 区块标题 |
| `items` | `array` | 职业经历数组 |
| `items[].period` | `string` | 时间段 |
| `items[].company` | `string` | 公司名称 |
| `items[].role` | `string` | 担任角色 |
| `items[].highlights` | `array` | 核心业绩亮点 |

**示例**：

```typescript
career: {

  items: [
    {
      period: '2022.04 - 至今',
      company: '某独角兽公司',

        '主导混合云容灾体系建设，将 RTO 从 2 小时缩短至 45 秒',
        '重构核心计费模块，支撑单日千万级订单',
      ],
    },
    // 可继续追加更多职业经历
  ],
}
```

## 5️⃣ 技能与工具箱（skills）

**说明**：以饼图形式展示技能分布，`categories` 数组可继续追加分类。

**配置项**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `title` | `string` | 区块标题 |
| `categories` | `array` | 技能分类数组 |
| `categories[].name` | `string` | 分类名称 |
| `categories[].items` | `array` | 技能项数组 |
| `items[].level` | `string` | 掌握程度文字 |
| `items[].levelClass` | `string` | 掌握程度样式：`master` / `proficient` / `plain` |
| `items[].label` | `string` | 技能描述 |

**示例**：

```typescript
skills: {
  title: '🛠️ 技能与工具箱',
  categories: [
    {
      name: '编程语言',
      items: [
        { level: '精通', levelClass: 'master', label: 'Java、Go（源码级）' },
        { level: '熟练', levelClass: 'proficient', label: 'Python、Shell（生产级）' },
      ],
    },
    // 可继续追加更多分类
  ],
}
```

**配置项**：

## 8️⃣ 底部联系信息（footer）

**说明**：设置页面底部联系方式和自定义图片。

**配置项**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `email` | `string` | 联系邮箱 |
| `replyNote` | `string` | 回复效率说明 |
| `updateText` | `string` | 页面更新时间 |
| `images` | `array` | 底部图片数组，支持字符串或对象 |

**图片支持三种形态**：

```typescript
// 本地路径
'/assets/img/fileName.png'

// 外部图片直链
'https://example.com/image.png'

// 可点击跳转的外部图片
{
  src: 'https://example.com/badge.png',
  alt: '徽章',
  href: 'https://example.com',
  external: true,
}
```

**图片布局规则**：

- 1 张：居中显示
- 2 张：左右并排
- 3 张：上 2 下 1，下方居中 → 三角布局
- 4 张：2×2 方阵
- 5 张：2 / 2 / 1，最后一行居中

**示例**：

```typescript
footer: {
  email: 'Misaka273@outlook.com',
  replyNote: '24 小时内必回',
  updateText: '更新于 2026 年 7 月',
  images: [
    '/assets/img/fileName.png',
    'https://example.com/image.png',
    {
      src: 'https://example.com/badge.png',
      alt: '徽章',
      href: 'https://example.com',
      external: true,
    },
  ] as FooterImageItem[],
}
```

## 9️⃣ 完整配置示例

### 用户自定义配置（推荐）

```typescript
// src/config/.config/about.config.ts
import type { AboutConfig, FooterImageItem } from '../about.config';

export const userAboutConfig: Partial<AboutConfig> = {
  hero: {
    name: '余温',
    handle: '@yu_wen',
    role: '资深云原生架构师 ｜ 技术专栏作者',
    valueProposition: '写看得懂的云原生实战笔记',
  },

  footer: {
    email: 'your@email.com',
    images: [
      '/assets/img/fileName.png',
    ] as FooterImageItem[],
  },
};

export default userAboutConfig;
```

## 💡 最佳实践

1. **不要直接修改默认配置**：在 `src/config/.config/about.config.ts` 中覆盖需要修改的字段
2. **图片路径**：本地图片放在 `public/assets/img/` 目录下
3. **外部图片**：确保外部链接可访问，必要时配置 `external: true`
4. **追加内容**：`story.paragraphs`、`career.items`、`skills.categories`、`influence.others` 都是数组，直接追加即可
5. **字段精简**：只需配置需要覆盖的字段，未配置的字段会沿用默认值

---

## ⚠️ 注意事项

1. 用户配置会**完全替换**对应数组，不会与默认数组合并
2. 如果用户配置中某个字段未提供，将使用默认配置中的值
3. 修改配置后需要重新构建才能生效
4. 底部图片一行最多显示两张，奇数时最后一张自动居中
