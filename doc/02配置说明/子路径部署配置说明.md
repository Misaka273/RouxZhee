# 🌐 子路径部署配置说明

本文档介绍如何将 RouxZhee 站点部署到域名的子路径下，例如 `zus.baimu.live/zr/`。

> 💡 **提示**：子路径部署不会影响默认的根路径部署方式，`npm run build` 仍然按原来的 `https://gl.baimu.live` 根路径构建。

---

## 1️⃣ 什么是子路径部署

子路径部署是指站点不放在域名的根目录 `/`，而是放在某个子目录下：

| 部署方式 | 访问地址 | 资源路径 |
|---------|---------|---------|
| 根路径部署 | `zus.baimu.live/` | `/assets/xxx` |
| 子路径部署 | `zus.baimu.live/zr/` | `/zr/assets/xxx` |

子路径部署后，页面内所有链接、图片、样式、脚本都需要自动加上 `/zr` 前缀，否则会出现 404。

---

## 2️⃣ 已内置的构建命令

项目已内置好子路径部署配置，只需执行：

```bash
npm run build:diylujing
```

这条命令会自动设置：

```bash
DEPLOY_BASE=/zr
DEPLOY_SITE=https://zus.baimu.live
```

<br>

package.json
完整示例

```
"build:diylujing": "set DEPLOY_BASE=/zr&& set DEPLOY_SITE=https://zus.baimu.live&& node scripts/rz-start.mjs build && npm run postbuild",
```

<br>

构建完成后，产物输出到 `dist/` 目录。

---

## 3️⃣ 修改子路径或域名

如果你想换成其他子路径或域名，修改 [`package.json`](file:///f:/RouxZhee/RouxZhee/package.json) 中的 `build:diylujing` 脚本：

```json
"build:diylujing": "set DEPLOY_BASE=/zr&& set DEPLOY_SITE=https://zus.baimu.live&& node scripts/rz-start.mjs build && npm run postbuild"
```

将 `/zr` 和 `https://zus.baimu.live` 替换为你自己的即可。

### 示例：部署到 /blog/

```json
"build:diylujing": "set DEPLOY_BASE=/blog&& set DEPLOY_SITE=https://mydomain.com&& node scripts/rz-start.mjs build && npm run postbuild"
```

### 手动指定环境变量

也可以直接通过环境变量构建：

```powershell
$env:DEPLOY_BASE="/zr"
$env:DEPLOY_SITE="https://zus.baimu.live"
npm run build
```

## 4️⃣ 服务器部署示例

构建完成后，把 `dist/` 目录下的所有文件上传到服务器的 `/zr/` 目录下。

### Nginx 配置

```nginx
server {
    listen 80;
    server_name zus.baimu.live;

    location /zr/ {
        alias /var/www/zus.baimu.live/zr/;
        index index.html;
    }
}
```

### Caddy 配置

```caddy
zus.baimu.live {
    handle_path /zr/* {
        root * /var/www/zus.baimu.live/zr
        file_server
    }
}
```

> ⚠️ **注意**：RouxZhee 是 Astro 生成的多页面静态站点，每个路由都有真实的 `index.html` 文件，因此不需要像单页应用（SPA）那样配置"所有路径回退到 index.html"。

---

## 5️⃣ 相关配置文件

子路径部署功能由以下文件共同实现：

| 文件 | 作用 |
|------|------|
| [`astro.config.mjs`](file:///f:/RouxZhee/RouxZhee/astro.config.mjs) | 读取 `DEPLOY_BASE` 和 `DEPLOY_SITE` 环境变量 |
| [`src/utils/base.ts`](file:///f:/RouxZhee/RouxZhee/src/utils/base.ts) | 提供 `withBase()` 工具函数，自动拼接子路径前缀 |
| [`package.json`](file:///f:/RouxZhee/RouxZhee/package.json) | 定义 `build:diylujing` 构建脚本 |
| [`scripts/pagefind-build.mjs`](file:///f:/RouxZhee/RouxZhee/scripts/pagefind-build.mjs) | 构建搜索索引 |
| [`public/sw.js`](file:///f:/RouxZhee/RouxZhee/public/sw.js) | Service Worker 自动根据注册 scope 推导子路径 |

## 6️⃣ 验证构建结果

构建完成后，可以检查生成的 HTML 中资源链接是否带上了子路径前缀：

```bash
# 查看首页 HTML 中的 /zr/ 链接
grep -o 'href="/zr/[^"]*"\|src="/zr/[^"]*"' dist/index.html | head -20
```

如果看到类似 `/zr/favicon.svg`、`/zr/_astro/...` 的输出，说明子路径配置已生效。

---

## 7️⃣ 常见问题

### Q1：子路径部署后图片 404？

请确保图片路径以 `/` 开头（如 `/assets/xxx.jpg`），系统会自动拼接 `/zr` 前缀。外部图片链接（如 `https://...`）不需要处理。

### Q2：搜索功能点击结果跳转 404？

搜索结果的 URL 会在运行时自动拼接子路径前缀。如果仍然 404，请确认是否正确执行了 `npm run build:diylujing`（包含 pagefind 索引生成）。

### Q3：开发环境如何预览子路径效果？

目前开发命令 `npm run dev` 仍然使用根路径。如需在开发环境预览子路径，可以临时修改 `astro.config.mjs` 中的 `base: '/zr/'`，开发完成后再改回环境变量方式。

### Q4：默认的根路径构建还能用吗？

可以，`npm run build` 仍然按原来的根路径 `https://gl.baimu.live` 构建，两种构建方式互不影响。

---

## 8️⃣ 总结

| 命令 | 用途 |
|------|------|
| `npm run build` | 根路径构建（默认） |
| `npm run build:diylujing` | 子路径 `/zr/` 构建 |

只需修改 `package.json` 中的 `DEPLOY_BASE` 和 `DEPLOY_SITE`，即可快速切换到任意子路径和域名部署。
